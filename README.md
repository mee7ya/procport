# procport

A Windows CLI tool that monitors active TCP connections for a given process.

## Features

- Tracks all TCP connections (IPv4 and IPv6) belonging to a process by name
- Live-refreshing table view in an alternate terminal screen
- Waits for the process to appear if it isn't running yet
- Optional connection history — dropped connections stay visible as `DROPPED`
- Optional loopback unwrapping — attempts to follow loopbacks to get actual connections

## Usage

```
procport [OPTIONS] <PROCESS_NAME>
```

`<PROCESS_NAME>` is the executable name, with or without `.exe` (e.g. `telegram.exe` or `telegram`).

### Options

| Flag | Description |
|---|---|
| `--preserve-history` | Keep dropped connections in the table, marked as `DROPPED` |
| `--unwrap-loopbacks` | Follow loopback connections to reveal real destination addresses |

### Examples

```powershell
# Monitor Process's TCP connections
procport process.exe

# Keep history of all connections seen
procport process.exe --preserve-history

# Unwrap loopback proxy to show real destinations
procport process.exe --unwrap-loopbacks
```

## How `--unwrap-loopbacks` works

Some processes route traffic through local proxies or helper processes (e.g. SOCKS proxies, VPN clients). In these cases the monitored process only shows connections to `127.0.0.1`, making the actual remote endpoints invisible.

With `--unwrap-loopbacks`, procport:

1. Finds all loopback remote addresses used by the target process.
2. Looks up which process is listening on each of those loopback ports.
3. Follows that process's own connections

The final table shows the real destinations instead of the intermediate loopback addresses.

> **Note:** Because this follows connections owned by *other* processes, it may occasionally surface addresses the monitored process isn't directly communicating with.

## Building

Requires Rust and a Windows host (uses Win32 APIs).

```powershell
cargo build --release
```

The binary will be at `target\release\procport.exe`.

## Platform

Windows only. Uses `GetExtendedTcpTable` and `CreateToolhelp32Snapshot` from the Win32 API.
